Storage SOC Timing in Peak Reserve Margin Calculation

Key Issue: When calculating unused storage capacity at the peak hour, it matters whether we use the State of Charge (SOC) before or after discharge during that hour.

How Storage SOC is Tracked in the Model

In the code, storageSOC[i] represents the State of Charge at the END of hour i (after any charging or discharging during that hour).

// From the code (lines 7400-7426)
for (let i = 0; i < 8760; i++) {
    const storageAction = hourlyGeneration['Storage'][i];
    // Positive = discharge, Negative = charge
    
    if (storageAction < 0) {
        // Charging: energy stored = charge amount * RTE
        currentSOC -= storageAction * conf.storageRTE;
    } else {
        // Discharging: energy removed directly
        currentSOC -= storageAction;
    }
    
    currentSOC = Math.max(0, Math.min(maxStorageEnergy, currentSOC));
    storageSOC[i] = currentSOC;  // SOC at END of hour i
}

Concrete Example

System Configuration

  • Storage Power Capacity: 200 MW
  • Storage Duration: 4 hours
  • Storage Energy Capacity: 200 MW × 4 hr = 800 MWh
  • Round-Trip Efficiency: 85%
  • Peak Load Hour: Hour 15
  • Peak Load: 1,000 MW

Hour-by-Hour Storage Operation

Hour Action Generation (MW) SOC Start (MWh) SOC End (MWh) Available Capacity (MW)
13 Charge -150 200 327.5 min(200, 200) = 200
14 Charge -100 327.5 412.5 min(200, 327.5) = 200
15 (PEAK) Discharge +150 412.5 262.5 min(200, 412.5) = 200
16 Discharge +100 262.5 162.5 min(200, 262.5) = 200
Note: In the model:
  • storageSOC[14] = 412.5 MWh (end of hour 14 = start of hour 15)
  • storageSOC[15] = 262.5 MWh (end of hour 15 = after discharge)

The Problem with Using Wrong SOC

❌ INCORRECT: Using SOC After Discharge

const storageSOC = modelResults.storageSOC[15]; // = 262.5 MWh (AFTER discharge) const storageCap = 200; // MW available = Math.min(storageCap, storageSOC); // = min(200, 262.5) = 200 MW const storageGen = 150; // MW discharged unused = available - storageGen; // = 200 - 150 = 50 MW

Why this seems okay but is conceptually wrong:

  • We're using the SOC after the discharge happened
  • The available capacity should be based on what was available before taking action
  • In this case, we got lucky because the min() capped at 200 MW anyway
  • But in edge cases (see below), this breaks completely

✅ CORRECT: Using SOC Before Discharge

// SOC at beginning of peak hour = SOC at end of previous hour const socBeforePeak = modelResults.storageSOC[14]; // = 412.5 MWh (BEFORE discharge) const storageCap = 200; // MW available = Math.min(storageCap, socBeforePeak); // = min(200, 412.5) = 200 MW const storageGen = 150; // MW discharged unused = available - storageGen; // = 200 - 150 = 50 MW

Why this is correct:

  • We're using the SOC at the beginning of the hour
  • This represents what was actually available to dispatch
  • The unused capacity is what we could have dispatched but didn't

Critical Edge Case: Fully Depleted Battery

Consider this scenario where the storage fully discharges at the peak hour:

Hour Generation (MW) SOC Start (MWh) SOC End (MWh)
15 +150 412.5 262.5
16 (PEAK) +200 262.5 62.5

❌ Using SOC After Discharge (WRONG)

available = min(200, 62.5) = 62.5 MW ← WRONG! discharged = 200 MW unused = 62.5 - 200 = -137.5 MW ← NONSENSE! Negative unused capacity!

✅ Using SOC Before Discharge (CORRECT)

available = min(200, 262.5) = 200 MW ← Correct! discharged = 200 MW unused = 200 - 200 = 0 MW ← Correct! Battery fully utilized

The Fix

The corrected code uses storageSOC[peakLoadHour - 1] to get the SOC at the beginning of the peak hour (which is the end of the previous hour):

// CORRECTED CODE:
if (tech.id === 'storage') {
    // Use SOC at beginning of peak hour (end of previous hour)
    const socBeforePeak = (modelResults.storageSOC && peakLoadHour > 0) 
        ? modelResults.storageSOC[peakLoadHour - 1] 
        : 0;  // If peak is hour 0, battery starts empty
    
    const storageCap = installedCapacities['storage'] || 0;
    availableDispatchableAtPeak += Math.min(storageCap, socBeforePeak);
    
    const storageGen = modelResults.hourlyGeneration['Storage'][peakLoadHour];
    actualDispatchAtPeak += Math.max(0, storageGen);
}

Summary

Key Takeaways:

  1. storageSOC[i] = SOC at end of hour i (after charging/discharging)
  2. To get available capacity at hour i, use storageSOC[i-1] (SOC at beginning)
  3. Available capacity = what you could have used before taking action
  4. Unused capacity = available - actually dispatched
  5. Using post-discharge SOC can lead to nonsensical negative unused capacity

Reserve Margin Interpretation:

The reserve margin measures the operating headroom you had at the critical hour. It answers the question: "How much more could we have dispatched if needed?" This requires knowing what was available before you dispatched, not after.
