Double Chart Rendering Fix Test

This test validates the fix for double chart rendering during batch year processing.

Problem Statement

The processYearOnMainThread function was causing double chart rendering:

  1. First render: processYearOnMainThread calls saveCurrentYear() which updates all 4 charts
  2. Second render: Worker sends year_complete message which calls updateMultiYearCharts()

This slowed down the batch processing unnecessarily.

Solution Implemented

  1. Added skipCharts option to saveCurrentYear() function
  2. Updated processYearOnMainThread() to pass skipCharts: true
  3. Created updateMultiYearCharts() helper function
  4. Charts now update only once per year (in worker message handler)
