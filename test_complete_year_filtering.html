Complete Year Filtering Test

Test Overview

This test validates that processHistoricalData() and calibrateFromZone() correctly filter data to include only complete years.

  • Problem: Previously, all rows (including partial years) were used for calibration, skewing the historicalAvgPrice
  • Fix: Now filters to complete years only during parsing
  • Expected: Only complete years are parsed (8,760 rows per year)

Test Scenarios

  1. Single Complete Year: Data with 8,760 rows for 2024 → All rows should be parsed
  2. Mixed Complete + Partial: 8,760 rows for 2024 + 4,380 rows for 2025 → Only 2024 rows should be parsed
  3. Multiple Complete Years: 8,760 rows each for 2024 and 2025 → All 17,520 rows should be parsed
