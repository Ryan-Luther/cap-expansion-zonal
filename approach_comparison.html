Two Approaches to Storage Reserve Margin

Question: When calculating unused storage capacity, should we:
  • Approach 1: Use SOC before discharge and subtract what was dispatched?
  • Approach 2: Use SOC after discharge and ignore what was dispatched?

Example Scenario: Peak at Hour 15

Parameter Value
Peak Load 1,000 MW
Storage Power Capacity 200 MW
Storage Energy Capacity 800 MWh
SOC at start of hour 15 (end of hour 14) 412.5 MWh
Storage discharge during hour 15 150 MW
SOC at end of hour 15 262.5 MWh
Thermal capacity 800 MW
Thermal dispatched 700 MW

The Two Approaches

Approach 1: SOC Before + Subtract Dispatch

Logic: Unused = What was available - What was used

// Storage calculation socBefore = storageSOC[14] = 412.5 MWh availableStorage = min(200, 412.5) = 200 MW dispatchedStorage = 150 MW unusedStorage = 200 - 150 = 50 MW // Total calculation availableThermal = 800 MW dispatchedThermal = 700 MW unusedThermal = 800 - 700 = 100 MW // Combined totalAvailable = 800 + 200 = 1000 MW totalDispatched = 700 + 150 = 850 MW totalUnused = 100 + 50 = 150 MW
Reserve Margin = 150 / 1000 = 15.0%

Interpretation: "We had 150 MW of dispatchable capacity that we didn't use at the peak hour."

Pros:

  • Consistent treatment of all resources (thermal + storage)
  • Answers: "How much more could we have dispatched?"
  • Matches typical reserve margin definition

Approach 2: SOC After + Ignore Dispatch

Logic: Unused = What capacity remains available

// Storage calculation socAfter = storageSOC[15] = 262.5 MWh availableStorage = min(200, 262.5) = 200 MW dispatchedStorage = 0 (ignored) unusedStorage = 200 - 0 = 200 MW // Total calculation availableThermal = 800 MW dispatchedThermal = 700 MW unusedThermal = 800 - 700 = 100 MW // Combined totalAvailable = 800 + 200 = 1000 MW totalDispatched = 700 + 0 = 700 MW totalUnused = 100 + 200 = 300 MW
Reserve Margin = 300 / 1000 = 30.0%

Interpretation: "We have 300 MW of dispatchable capacity still available (including storage's remaining capability)."

Pros:

  • Simpler calculation (no need to use previous hour SOC)
  • Represents instantaneous available capacity
  • Storage dispatch is already "baked into" the SOC value

Which Interpretation is Correct?

Key Question: What does "unused capacity" mean?

Option A: Capacity we could have used but didn't

  • This matches Approach 1
  • Storage: Had 200 MW available, used 150 MW, unused = 50 MW
  • Answers: "How much additional capacity could we have dispatched?"

Option B: Capacity that remains available after dispatch

  • This matches Approach 2
  • Storage: After using some, still have 200 MW available (limited by SOC)
  • Answers: "How much capacity do we have right now?"

Edge Case: Storage Nearly Depleted

Scenario: Peak at hour 16, storage fully discharges 200 MW

  • SOC before (end of hour 15): 262.5 MWh
  • Discharge during hour 16: 200 MW
  • SOC after (end of hour 16): 62.5 MWh

Approach 1

availableStorage = min(200, 262.5) = 200 MW dispatchedStorage = 200 MW unusedStorage = 200 - 200 = 0 MW

Result: Storage was fully utilized ✓

Approach 2

availableStorage = min(200, 62.5) = 62.5 MW dispatchedStorage = 0 (ignored) unusedStorage = 62.5 MW

Result: Storage has 62.5 MW remaining capacity available

My Recommendation

Both approaches are mathematically valid, but they answer different questions:

  • Approach 1 (current implementation): Better for measuring operating reserves - how much headroom you had at the critical hour
  • Approach 2 (your suggestion): Better for measuring remaining capacity - what you could dispatch in the next hour

For "Peak Reserve Margin" as typically defined in reliability studies, Approach 1 is more standard because it measures spare capacity that was available but not used during the peak event.

However, Approach 2 is valid if you want to measure the instantaneous reserve capability at the end of the peak hour, which could be useful for understanding system resilience.
